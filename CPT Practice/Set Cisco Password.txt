Below is the **correct CCNA-level full workflow** to set **ALL passwords in encrypted form** on a Cisco switch **and make them permanent**.

---

## 🔐 Encrypted Passwords – Full Workflow (Step by Step)

### 1️⃣ Enter privileged mode

```
Switch>enable
```

---

### 2️⃣ Go to global configuration mode

```
Switch#configure terminal
```

---

### 3️⃣ Set **encrypted enable password** (MOST IMPORTANT)

✅ This is already encrypted by default

```
Switch(config)#enable secret ccna2
```

> ⚠️ **Do NOT use `enable password`**
> `enable secret` is always encrypted and overrides it.

---

### 4️⃣ Encrypt ALL other passwords

This encrypts console & vty passwords stored in config.

```
Switch(config)#service password-encryption
```

---

### 5️⃣ Set **console password (encrypted)**

```
Switch(config)#line console 0
Switch(config-line)#password cisco
Switch(config-line)#login
Switch(config-line)#exit
```

➡ Password will be encrypted automatically because of
`service password-encryption`

---

### 6️⃣ Set **VTY (Telnet) password (encrypted)**

```
Switch(config)#line vty 0 4
Switch(config-line)#password class
Switch(config-line)#login
Switch(config-line)#exit
```

---

### 7️⃣ (Recommended) Disable DNS lookup

Prevents this error:
`Translating "end"...`

```
Switch(config)#no ip domain-lookup
```

---

### 8️⃣ Exit configuration mode

```
Switch(config)#end
```

---

### 9️⃣ 💾 SAVE the configuration (VERY IMPORTANT)

If you skip this → passwords will be lost ❌

```
Switch#copy running-config startup-config
```

OR

```
Switch#write memory
```

---

## ✅ Verify encryption

```
Switch#show running-config
```

You should see:

* `enable secret 5 ...`  (hashed)
* Console & vty passwords scrambled (not readable)

Example:

```
enable secret 5 $1$kfj3$9sdf...
password 7 0822455D0A16
```

Great — this is a **perfect practice config** 👍
I’ll explain it **line by line**, **what it does**, and **where each password is actually set**.

---

## 1️⃣ `enable`

**What it does**

* Moves you from **user EXEC mode** to **privileged EXEC mode**

```
Switch>  →  Switch#
```

🔑 **Password used here**

* This is where the **enable secret password** will be required **next time** you type `enable`.

---

## 2️⃣ `configure terminal`

**What it does**

* Enters **global configuration mode**

```
Switch#  →  Switch(config)#
```

No password involved here (you’re already privileged).

---

## 3️⃣ `enable secret ccna2`

**What it does**

* Sets the **privileged (enable) mode password**
* This password is **encrypted automatically**

🔐 **Password set**

* **Enable mode password** = `ccna2`

📍 **Where it is stored**

```
enable secret 5 <encrypted-hash>
```

📌 Used when:

```
Switch> enable
Password: ccna2
```

---

## 4️⃣ `service password-encryption`

**What it does**

* Encrypts **all plain-text passwords** in the configuration
* Affects:

  * Console password
  * VTY (Telnet) passwords

🔐 **No password is set here**

* This command only **encrypts**, it does NOT create passwords.

---

## 5️⃣ `line console 0`

**What it does**

* Enters **console line configuration mode**
* This is the physical cable login (PC → switch)

```
Switch(config)# → Switch(config-line)#
```

---

## 6️⃣ `password cisco`

**What it does**

* Sets the **console access password**

🔐 **Password set**

* **Console password** = `cisco`

📍 **Where it is stored**

```
line console 0
 password 7 <encrypted-value>
```

(Encrypted because of `service password-encryption`)

---

## 7️⃣ `login`

**What it does**

* Tells the switch:

  > “Ask for the password when someone connects via console”

❌ Without this → password is ignored.

---

## 8️⃣ `exit`

**What it does**

* Exits console line mode back to global config

---

## 9️⃣ `no ip domain-lookup`

**What it does**

* Stops the switch from trying DNS when you mistype a command

🚫 No password involved.

---

## 🔟 `end`

**What it does**

* Exits configuration mode back to privileged EXEC

---

## 1️⃣1️⃣ `copy running-config startup-config`

**What it does**

* Saves **everything** to NVRAM
* Makes passwords **permanent**

💾 Without this → passwords disappear on reboot.

---

| ------------------------ | --------------------------------------- | --------- |
| Enable (privileged mode) | `enable secret ccna2`                   | **ccna2** |
| Console login            | `password cisco` (under line console 0) | **cisco** |
